개요(Overview)

Why Sass?! CSS의 확장 언어인 Sass. CSS를 사용하다보면 그야말로 한땀한땀 하드코딩을 해나가야 합니다. 물론 하드코딩 손 맛~이라는게 있지만, 프로젝트가 커지면... 후아후아~
Sass는 변수와 함수등의 개념을 도입함으로서 좀 더 깔끔한 코딩, 유지보수가 좋은 코딩을 할 수 있도록 도와줍니다.
특히 공통으로 반복되는 코드 관리, 자동으로 연산이 가능하게 해주는 기능만 보더라도 왜 Sass를 사용하면 편리한지 알게해주죠. 아 멋짐~!!
Front-End 개발의 길을 걷게 된 이상, 새로운 분야에 대한 공부는 즐거움으로 받아들이기로 했습니다. 하... ^^

목표

HTML, CSS로 개발한 반응형웹 사이트를 Sass로 Code Refactoring해보기(Code Refactoring : 코드의 가독성이 낮거나 중복된 로직 등 제거. 동작상태에 대한 변화없이 코드의 스타일을 변경시키는 작업)

Project 공통 고려사항
  • 웹표준 준수
  • 웹접근성 고려(음성출력/키보드 접근 등)
  • HTML 올바른 의미, 구조화
  • naming 규칙. 간결하며 읽기 용이하게
  • 코드 최적화 : 불필요한 <div>, <span>요소의 남용 등, 중복되는 스타일 코드없애기, 코드는 읽기 쉽고 간결하게 작성
  • Sass 스타일 설계, 모듈화
반응형 웹 고려사항

Mobile First : 모바일 화면에 맞는 UI를 우선시하여 설계 한 다음 데스크탑으로 점진적 향상시킵니다.
